# 灵鉴孔位培养扫描交接文档

> 实现 `SlotCultivationReader`，自动采集「灵鉴」界面各区域每个孔位的培养加分（cultivation_score），为后续「山河器最优摆放策略计算」提供数据基础。

## 1. 工作路径

### 工具使用统计
| 工具名称 | 调用次数 | 主要用途 |
|---------|---------|---------|
| Read | 约 30 次 | 读取 `lingjian_topology.json`、现有交接文档格式、`SlotCultivationScanner`、面板截图、单元测试 |
| Write | 约 8 次 | 新建 `SlotCultivationScanner`、`LingjianNavigator`、`SlotCultivationReader`、模板图片、交接文档 |
| Edit | 约 25 次 | 实现模板匹配编号识别、修复下拉框滚动方向、调整测试用例、集成 CLI |
| Bash | 约 30 次 | 裁剪模板、离线验证、真实环境扫描、单元测试、滚动调试 |

### 执行阶段
- **阶段一：骨架与拓扑**
  - 目标：建立区域/孔位拓扑模型与校准加载器。
  - 关键动作：
    - 创建 `shq/scanner/topology_loader.py`，将 `data/lingjian_topology.json` 加载为 `Lingjian` 模型 + `RegionCalibration`。
    - 拓扑只记录每个区域有哪些孔位编号（1~N），不记录动态加分。

- **阶段二：导航器实现**
  - 目标：让程序能自动切换到灵鉴的任意区域并打开「孔位培养」面板。
  - 关键动作：
    - 创建 `shq/scanner/lingjian_navigator.py`，复用 `WukuNavigator` 导航到灵鉴。
    - 实现左侧区域下拉框展开/选择/滚动，支持未解锁区域检测。
    - 实现「孔位培养」标签页点击与面板开关。

- **阶段三：编号识别（图像方案）**
  - 目标：准确识别面板上艺术字/发光的孔位编号（壹、贰、叁……）。
  - 关键动作：
    - 先尝试 OCR，发现对 stylized 大写数字识别率极低。
    - 改用 Canny 方块检测，但中心圆盘装饰导致孔位数量误判。
    - 最终采用**模板匹配**：为 2~6 孔位区域分别裁剪 `壹~陆` 模板，按孔位数加载对应模板集并做多尺度匹配。

- **阶段四：加分读取**
  - 目标：把每个编号下方的「孔位评分：+XXXX」正确归到对应孔位。
  - 关键动作：
    - OCR 识别面板内所有含「孔位评分」或「评分」+ 数字的文字。
    - 对每个编号位置，找最近的加分文字；未找到则 cultivation_score 记 0。

- **阶段五：真实环境调试**
  - 目标：在真实游戏窗口下跑通端到端扫描。
  - 关键动作：
    - 发现前四个区域被误判为未解锁，定位到下拉框滚轮方向反了且滚动位置偏右。
    - 修正 `_scroll_dropdown` 的滚动坐标与方向逻辑。
    - 重新扫描，6 个区域全部成功，无低置信告警。

- **阶段六：回归验证**
  - 目标：确保代码改动不破坏既有功能。
  - 关键动作：
    - 单元测试：`python -m pytest tests/ -q` 69 个全部通过。
    - 真实扫描结果与用户手动读数一致。

## 2. 关键发现

### 任务范围内的发现
- 孔位编号是艺术字/大写数字，普通 OCR 几乎无法识别；模板匹配是可靠方案。
- 同一数字在不同孔位数区域中大小、位置不同，需要**按孔位数分子目录维护模板**。
- 编号存在激活（红色发光）和未激活（灰色）两种状态，模板集应包含多种状态变体以避免误配。
- 纯黑/纯色背景会对模板匹配产生高置信度误检，加入**边缘密度校验**后可有效剔除。
- 区域下拉框可滚动，但滚轮方向与普通列表相反：要显示上方区域需**向上滚动**，显示下方区域需**向下滚动**。
- 下拉框滚动鼠标必须位于列表文字区域（约 x=0.15w），过于偏右会失效。

### 延伸发现
- 灵鉴面板左侧布局固定，每个区域的孔位数量固定，只是具体加分因人而异；静态拓扑 + 动态读取是正确的拆分。
- 培养面板打开后右侧会显示单个孔位详情，但左侧主面板已包含所有编号与加分，无需逐个点击孔位。

## 4. 核心结论

本次任务完成了**灵鉴孔位培养加分的自动采集**：

1. **编号识别鲁棒**：通过按孔位数维护的模板集 + 多尺度匹配 + 边缘密度校验，稳定识别艺术字编号。
2. **区域切换可靠**：修正下拉框滚动方向与位置后，6 个区域均可正常切换，未解锁判断不再误判。
3. **加分读取准确**：以当前用户数据为例，扫描结果与用户手动读数一致：`长烟烽火` 孔位 1/2 各 +3600，`关河道远` 孔位 1/2 各 +3600，其余为 0。
4. **低置信情况清零**：本次正式扫描 `low_confidence` 为 0，`locked_regions` 为空。

## 6. 假设前提

- 游戏窗口客户区固定为 1334×750；所有 ROI 与模板基于此分辨率标定。
- 区域下拉列表顺序保持：驿寄梅花、戍客怀归、长烟烽火、黄泉夜渡、关河道远、骸关断云。
- 用户已安装 `rapidocr-onnxruntime` 并使用 RapidOCR 后端。
- 孔位编号字体/样式未发生大幅改版；若改版需重新裁剪模板。
- 用户理解自动化点击存在封号风险，自愿承担。
